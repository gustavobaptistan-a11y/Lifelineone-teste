# Manutencao

Antes de alterar o projeto:

1. identificar o modulo dono do comportamento;
2. verificar usos e testes relacionados;
3. formular uma hipotese verificavel;
4. fazer a menor alteracao possivel;
5. executar validacao focada;
6. atualizar a documentacao afetada.

Nao versionar `.env`, bancos locais, binarios ou ambientes virtuais. Rotacionar credenciais quando houver qualquer exposicao.
